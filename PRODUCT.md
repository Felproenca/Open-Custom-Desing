# Product

## Register

product

## Users

Developers and designers who need to create design artifacts (prototypes, landing pages, dashboards, decks, mobile apps) using AI-powered coding agents. They work in a local-first environment with real file access, wanting professional-grade output that looks intentional and crafted, not generic.

## Product Purpose

Open Design is an open-source alternative to Claude Design that connects existing coding agent CLIs to a skill-driven design workflow. It generates production-ready design artifacts through a deterministic pipeline: brief → discovery → direction → artifact → critique → export. The goal is to deliver design quality comparable to professional designers, but powered by AI agents running locally.

## Brand Personality

Creative, expressive, bold. The interface should feel like a powerful design tool that enables extraordinary creative output — not a toy, not a corporate dashboard, but a serious instrument for visual creation.

## Anti-references

- Overly playful/toy-like aesthetics — avoid childish or overly casual visual treatment
- Generic SaaS dashboards — avoid bland, template-like corporate designs
- Dated skeuomorphism — avoid old-fashioned realistic textures and shadows
- Cream/sand/beige body backgrounds — the saturated AI default of 2026
- Gradient text, glassmorphism as default, hero-metric templates
- Identical card grids with icon + heading + text
- Tiny uppercase tracked eyebrows above every section

## Design Principles

1. **Craft over convention** — Every design choice should feel intentional and considered, not assembled from templates
2. **Retro-modern fusion** — Blend nostalgic design elements (70s/80s aesthetic) with modern technical precision
3. **Expressive functionality** — Visual personality should enhance, not compromise, usability and clarity
4. **Deterministic beauty** — Design systems and palettes should produce consistent, high-quality output
5. **Local-first authenticity** — The tool runs on the user's machine; the design should feel grounded and real, not cloud-generic

## Accessibility & Inclusion

No specific WCAG requirements. Focus on general usability and visual clarity. Support reduced motion preferences as a courtesy, not a compliance requirement.
