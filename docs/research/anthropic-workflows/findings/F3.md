# F3: Multi-agent orchestration with Claude

## Findings

### [1] Anthropic distinguishes between workflows (predefined orchestration) and agents (dynamic self-directed systems)
- quote: "Workflows are systems where LLMs and tools are orchestrated through predefined code paths. Agents, on the other hand, are systems where LLMs dynamically direct their own processes and tool usage, maintaining control over how they accomplish tasks."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: 2024-12-19
- confidence: high

### [2] The orchestrator-workers pattern uses a central LLM to dynamically delegate tasks to worker LLMs
- quote: "In the orchestrator-workers workflow, a central LLM dynamically breaks down tasks, delegates them to worker LLMs, and synthesizes their results."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: 2024-12-19
- confidence: high

### [3] Claude Agent SDK supports subagents via the `agents` parameter for context isolation and parallelization
- quote: "Subagents are separate agent instances that your main agent can spawn to handle focused subtasks. Use them to isolate context, run multiple analyses in parallel, and apply specialized instructions without adding to the main agent's prompt."
- url: https://platform.claude.com/docs/en/agent-sdk/subagents
- source_type: primary
- published: unknown
- confidence: high

### [4] Subagents run in fresh context with only the final message returned to parent
- quote: "Each subagent runs in its own fresh conversation. Intermediate tool calls and results stay inside the subagent; only its final message returns to the parent."
- url: https://platform.claude.com/docs/en/agent-sdk/subagents
- source_type: primary
- published: unknown
- confidence: high

### [5] AgentDefinition allows configuring per-subagent prompts, tools, models, and permissions
- quote: "Each subagent can have tailored system prompts with specific expertise, best practices, and constraints. Subagents can be limited to specific tools, reducing the risk of unintended actions."
- url: https://platform.claude.com/docs/en/agent-sdk/subagents
- source_type: primary
- published: unknown
- confidence: high

### [6] Subagents can run in background mode for non-blocking parallel execution
- quote: "Subagents run in the background by default. An Agent tool call that omits the run_in_background input launches a background subagent, and Claude sets run_in_background: false when it needs the result before continuing."
- url: https://platform.claude.com/docs/en/agent-sdk/subagents
- source_type: primary
- published: unknown
- confidence: high

### [7] Nested subagents are supported up to 5 levels deep
- quote: "As of Claude Code v2.1.172, subagents can spawn their own subagents. A subagent five levels below the main agent can't spawn further subagents."
- url: https://platform.claude.com/docs/en/agent-sdk/subagents
- source_type: primary
- published: unknown
- confidence: high

### [8] Subagents can be resumed to continue previous work with full context
- quote: "You can resume a subagent to continue where it left off rather than starting fresh. A resumed subagent retains its full conversation history, including all previous tool calls, results, and reasoning."
- url: https://platform.claude.com/docs/en/agent-sdk/subagents
- source_type: primary
- published: unknown
- confidence: high

### [9] The evaluator-optimizer pattern enables iterative refinement between generator and evaluator LLMs
- quote: "In the evaluator-optimizer workflow, one LLM call generates a response while another provides evaluation and feedback in a loop."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: 2024-12-19
- confidence: high

### [10] For large-scale orchestration (dozens to hundreds of agents), use the Workflow tool instead of subagents
- quote: "For runs that coordinate dozens to hundreds of agents, use the Workflow tool, which moves the orchestration into a script the runtime executes outside the conversation context."
- url: https://platform.claude.com/docs/en/agent-sdk/subagents
- source_type: primary
- published: unknown
- confidence: high

### [11] Claude can automatically invoke subagents based on task description matching
- quote: "Claude automatically decides when to invoke subagents based on the task and each subagent's description. For example, if you define a performance-optimizer subagent with the description 'Performance optimization specialist for query tuning', Claude will invoke it when your prompt mentions optimizing queries."
- url: https://platform.claude.com/docs/en/agent-sdk/subagents
- source_type: primary
- published: unknown
- confidence: high

### [12] Anthropic recommends starting simple and adding multi-agent complexity only when measurably beneficial
- quote: "Success in the LLM space isn't about building the most sophisticated system. It's about building the right system for your needs. Start with simple prompts, optimize them with comprehensive evaluation, and add multi-step agentic systems only when simpler solutions fall short."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: 2024-12-19
- confidence: high
