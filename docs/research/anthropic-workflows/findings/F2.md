# F2: Tool use and MCP in Anthropic's ecosystem

## Findings

### [1] Claude's tool use follows a client/server architecture: client tools execute in your application, server tools execute on Anthropic infrastructure
- quote: "Client tools run in your application. Claude responds with stop_reason: 'tool_use' and one or more tool_use blocks. Your code executes the operation and sends back a tool_result. Server tools run on Anthropic's infrastructure: you see the results directly without handling execution."
- url: https://docs.anthropic.com/en/docs/build-with-claude/tool-use/overview
- source_type: primary
- published: 2025 (current docs)
- confidence: high

### [2] The Model Context Protocol (MCP) is an open-source JSON-RPC 2.0 standard for connecting AI applications to external tools, resources, and prompts
- quote: "MCP (Model Context Protocol) is an open-source standard for connecting AI applications to external systems. Think of MCP like a USB-C port for AI applications."
- url: https://modelcontextprotocol.io/introduction
- source_type: primary
- published: 2025 (MCP spec version 2025-06-18)
- confidence: high

### [3] Anthropic's MCP Connector enables direct Messages API integration with remote MCP servers without implementing an MCP client, currently in beta
- quote: "Claude's Model Context Protocol (MCP) connector feature enables you to connect to remote MCP servers directly from the Messages API without a separate MCP client. This feature requires the beta header: 'anthropic-beta': 'mcp-client-2025-11-20'"
- url: https://docs.anthropic.com/en/docs/agents-and-tools/mcp-connector
- source_type: primary
- published: 2025-11-20 (beta header)
- confidence: high

### [4] MCP defines three server-side primitives: Tools (executable functions), Resources (data sources), and Prompts (reusable templates)
- quote: "MCP defines three core primitives that servers can expose: Tools: Executable functions that AI applications can invoke to perform actions. Resources: Data sources that provide contextual information to AI applications. Prompts: Reusable templates that help structure interactions with language models."
- url: https://modelcontextprotocol.io/docs/learn/architecture
- source_type: primary
- published: 2025-06-18 (spec version)
- confidence: high

### [5] The tool search tool scales to 10,000+ tools by letting Claude search on-demand with regex or BM25, reducing context bloat by over 85%
- quote: "Loading every tool definition up front causes two problems as a tool library grows: Context bloat: A typical multi-server setup can consume ~55k tokens... Tool search typically reduces this by over 85 percent, loading only the 3-5 tools Claude needs for a given request. Tool selection accuracy: Claude's ability to pick the right tool degrades once you exceed 30-50 available tools."
- url: https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/tool-search-tool
- source_type: primary
- published: 2025-11-19 (tool version)
- confidence: high

### [6] Structured outputs guarantee schema-compliant JSON via constrained decoding and also power strict tool use (strict: true) for tool input validation
- quote: "Structured outputs constrain Claude's responses to follow a specific schema, ensuring valid, parseable output for downstream processing. Structured outputs provide two complementary features: JSON outputs (output_config.format) and Strict tool use (strict: true)."
- url: https://docs.anthropic.com/en/docs/build-with-claude/structured-outputs
- source_type: primary
- published: 2025 (current docs)
- confidence: high

### [7] Anthropic identifies five core agentic workflow patterns: prompt chaining, routing, parallelization, orchestrator-workers, and evaluator-optimizer
- quote: "We categorize all these variations as agentic systems, but draw an important architectural distinction between workflows and agents: Workflows are systems where LLMs and tools are orchestrated through predefined code paths. Agents, on the other hand, are systems where LLMs dynamically direct their own processes and tool usage."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [8] Programmatic tool calling lets Claude write code that calls tools from within a code execution container, improving performance by ~11% while using 24% fewer tokens on agentic search benchmarks
- quote: "On agentic search benchmarks like BrowseComp and DeepSearchQA, adding programmatic tool calling on top of basic search tools improved performance by an average of 11% while using 24% fewer input tokens."
- url: https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/programmatic-tool-calling
- source_type: primary
- published: 2026-01-20 (code execution version)
- confidence: high

### [9] Anthropic provides Anthropic-schema client tools (Bash, Text Editor, Computer Use, Memory) where Anthropic publishes the schema but your application executes the calls
- quote: "Anthropic-schema client tools: Anthropic publishes the schema and trains Claude on it. Your application still executes each call and returns the tool_result."
- url: https://docs.anthropic.com/en/docs/build-with-claude/tool-use/overview
- source_type: primary
- published: 2025 (current docs)
- confidence: high

### [10] The MCP Connector supports allowlisting, denylisting, and per-tool deferred loading configuration across multiple remote MCP servers in a single API request
- quote: "Key features: Direct API integration, Tool calling support, Flexible tool configuration (allowlist specific tools, or denylist unwanted tools), Per-tool configuration, Multiple servers in a single request."
- url: https://docs.anthropic.com/en/docs/agents-and-tools/mcp-connector
- source_type: primary
- published: 2025-11-20 (beta)
- confidence: high

### [11] Anthropic advises against complex agentic frameworks, recommending simple composable patterns with direct LLM API usage for most applications
- quote: "Consistently, the most successful implementations weren't using complex frameworks or specialized libraries. Instead, they were building with simple, composable patterns... We suggest that developers start by using LLM APIs directly: many patterns can be implemented in a few lines of code."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [12] MCP uses stateful connections with capability negotiation, and supports both STDIO (local) and Streamable HTTP (remote) transports
- quote: "MCP consists of two layers: Data layer (JSON-RPC 2.0 based protocol) and Transport layer (STDIO for local processes, Streamable HTTP for remote servers). MCP is a stateful protocol that requires lifecycle management."
- url: https://modelcontextprotocol.io/docs/learn/architecture
- source_type: primary
- published: 2025-06-18
- confidence: high
