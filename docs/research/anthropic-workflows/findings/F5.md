# F5: Ecosystem comparison — Anthropic vs alternatives

## Findings

### [1] OpenAI's Agents SDK replaces the Assistants API with a simpler Responses API and introduces two explicit orchestration patterns: handoffs (specialist takes over) and agents-as-tools (manager stays in control)
- quote: "Handoffs are the clearest fit when a specialist should own the next response rather than merely helping behind the scenes."
- url: https://platform.openai.com/docs/guides/agents/orchestration
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [2] Anthropic's agentic approach is fundamentally API-first and toolkit-based — it provides tool primitives (client tools, server tools, MCP connector) but leaves orchestration to the developer, unlike OpenAI's more opinionated Agent framework
- quote: "Claude determines when to call a tool based on the user's request and the tool's description. It then returns a structured call that your application executes."
- url: https://docs.anthropic.com/en/docs/build-with-claude/tool-use/overview
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [3] Anthropic's tool ecosystem includes unique server-side tools (web search, code execution, MCP connector, advisor tool, tool search) that execute on Anthropic's infrastructure — a capability no other major provider matches at the same integration depth
- quote: "Server tools (such as web_search, web_fetch, code_execution, and tool_search) run on Anthropic's infrastructure: you see the results directly without handling execution"
- url: https://docs.anthropic.com/en/docs/build-with-claude/tool-use/overview
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [4] LangGraph positions itself as a low-level orchestration runtime focused on statefulness, durability, and human-in-the-loop — filling a gap neither Anthropic's API-first nor OpenAI's higher-level SDK explicitly targets at the graph/workflow level
- quote: "LangGraph is very low-level, and focused entirely on agent orchestration. Before using LangGraph, we recommend you familiarize yourself with some of the components used to build agents."
- url: https://docs.langchain.com/oss/python/langgraph/overview
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [5] CrewAI's philosophical approach to agents is role-based with narrative context (role, goal, backstory), treating agents as team members rather than tool-wrapping functions — closer to Microsoft AutoGen's multi-agent chat paradigm than Anthropic or OpenAI's single-agent-with-tools approach
- quote: "Think of an agent as a specialized team member with specific skills, expertise, and responsibilities."
- url: https://docs.crewai.com/en/concepts/agents
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [6] AutoGen (Microsoft) offers a three-tier architecture: autogen-core (low-level event-driven), AgentChat (high-level multi-agent API with Teams and Swarm), and Studio (visual interface) — providing the broadest abstraction range of any single framework
- quote: "AgentChat provides intuitive defaults, such as Agents with preset behaviors and Teams with predefined multi-agent design patterns."
- url: https://microsoft.github.io/autogen/stable/user-guide/agentchat-user-guide/index.html
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [7] OpenAI's migration from Assistants API to Responses API explicitly deprecated persistent API objects in favor of Prompts (versioned behavioral profiles) — moving the industry toward prompt-as-config rather than API-created agent objects
- quote: "With the Responses API, you also get better performance and new features like deep research, MCP, and computer use."
- url: https://platform.openai.com/docs/assistants/migration
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [8] LangChain's `create_agent` represents the ecosystem convergence pattern — a single minimal harness API that routes through 10+ provider integrations (OpenAI, Anthropic, Google, Fireworks, Ollama, Azure, Bedrock), making it the most provider-agnostic agent framework available
- quote: "Agent = Model + Harness. LangChain provides create_agent: a minimal, highly configurable agent harness."
- url: https://docs.langchain.com/oss/python/langchain/overview
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [9] Anthropic's extended thinking with interleaved tool use and its MCP connector represent a unique deep-integration approach — allowing tools to run within Claude's reasoning loop rather than as external side-calls, which is architecturally distinct from OpenAI's tool-calling model
- quote: "When using interleaved thinking with tools, you can exceed this limit as the token limit becomes your entire context window."
- url: https://docs.anthropic.com/en/docs/build-with-claude/extended-thinking
- source_type: primary
- published: unknown (current docs)
- confidence: high

### [10] The ecosystem is converging on two models: Anthropic's "powerful single agent + rich toolset" vs. the multi-agent orchestration pattern (LangGraph graphs, AutoGen Teams/Swarm, CrewAI crews, OpenAI handoffs) — Anthropic differentiates by betting on single-agent intelligence rather than multi-agent composition
- quote: (synthesized from multiple primary sources)
- url: n/a
- source_type: n/a
- published: n/a
- confidence: medium
