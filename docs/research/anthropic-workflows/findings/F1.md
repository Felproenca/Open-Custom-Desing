# F1: Core concepts and taxonomy of Anthropic workflow patterns

## Findings

### [1] Anthropic distinguishes "workflows" (orchestrated through predefined code paths) from "agents" (LLMs dynamically directing their own processes and tool usage)
- quote: "Workflows are systems where LLMs and tools are orchestrated through predefined code paths. Agents, on the other hand, are systems where LLMs dynamically direct their own processes and tool usage, maintaining control over how they accomplish tasks."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [2] The foundational building block is the "augmented LLM" — an LLM enhanced with retrieval, tools, and memory
- quote: "The basic building block of agentic systems is an LLM enhanced with augmentations such as retrieval, tools, and memory. Our current models can actively use these capabilities—generating their own search queries, selecting appropriate tools, and determining what information to retain."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [3] Anthropic identifies exactly five workflow patterns: prompt chaining, routing, parallelization, orchestrator-workers, and evaluator-optimizer
- quote: "We'll start with our foundational building block—the augmented LLM—and progressively increase complexity, from simple compositional workflows to autonomous agents."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [4] Prompt chaining decomposes a task into sequential steps, trading latency for higher accuracy by making each LLM call an easier task
- quote: "Prompt chaining decomposes a task into a sequence of steps, where each LLM call processes the output of the previous one. You can add programmatic checks (see 'gate' in the diagram below) on any intermediate steps to ensure that the process is still on track."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [5] Routing classifies an input and directs it to a specialized followup task, enabling separation of concerns and more specialized prompts
- quote: "Routing classifies an input and directs it to a specialized followup task. This workflow allows for separation of concerns, and building more specialized prompts. Without this workflow, optimizing for one kind of input can hurt performance on other inputs."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [6] Parallelization has two key variations: sectioning (breaking a task into independent parallel subtasks) and voting (running the same task multiple times for diverse outputs)
- quote: "This workflow, parallelization, manifests in two key variations: Sectioning: Breaking a task into independent subtasks run in parallel. Voting: Running the same task multiple times to get diverse outputs."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [7] The orchestrator-workers pattern differs from parallelization in its flexibility — subtasks aren't pre-defined but determined by the orchestrator based on the specific input
- quote: "Whereas it's topographically similar, the key difference from parallelization is its flexibility—subtasks aren't pre-defined, but determined by the orchestrator based on the specific input."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [8] Evaluator-optimizer uses one LLM to generate and another to evaluate in a loop, effective when clear evaluation criteria exist and iterative refinement provides measurable value
- quote: "This workflow is particularly effective when we have clear evaluation criteria, and when iterative refinement provides measurable value. The two signs of good fit are, first, that LLM responses can be demonstrably improved when a human articulates their feedback; and second, that the LLM can provide such feedback."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [9] Anthropic's three core principles for agent design: simplicity, transparency, and well-crafted agent-computer interfaces (ACI)
- quote: "When implementing agents, we try to follow three core principles: 1. Maintain simplicity in your agent's design. 2. Prioritize transparency by explicitly showing the agent's planning steps. 3. Carefully craft your agent-computer interface (ACI) through thorough tool documentation and testing."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [10] Anthropic recommends starting with simple solutions and only increasing complexity when needed; for many applications, single LLM calls with retrieval and in-context examples suffice
- quote: "When building applications with LLMs, we recommend finding the simplest solution possible, and only increasing complexity when needed. This might mean not building agentic systems at all."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [11] Agents are best for open-ended problems where the required number of steps can't be predicted, with higher costs and potential for compounding errors requiring sandboxed testing
- quote: "Agents can be used for open-ended problems where it's difficult or impossible to predict the required number of steps, and where you can't hardcode a fixed path. The LLM will potentially operate for many turns, and you must have some level of trust in its decision-making."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high

### [12] Anthropic spent more time optimizing tool definitions (ACI) than the overall prompt when building their SWE-bench agent, including changing tools to require absolute filepaths to prevent model errors
- quote: "While building our agent for SWE-bench, we actually spent more time optimizing our tools than the overall prompt. For example, we found that the model would make mistakes with tools using relative filepaths after the agent had moved out of the root directory. To fix this, we changed the tool to always require absolute filepaths."
- url: https://www.anthropic.com/engineering/building-effective-agents
- source_type: primary
- published: Dec 19, 2024
- confidence: high
